# 숫자 관련

### 1. `Number()` 함수 사용

`Number()` 함수를 사용하여 문자열을 숫자로 변환할 수 있습니다.

```typescript
const str: string = "123";
const num: number = Number(str);
console.log(num); // 123
```

### 2. `parseInt()` 함수 사용
`parseInt()` 함수는 문자열을 정수로 변환합니다. 두 번째 인자로 `radix`(기수)를 넣어 진법을 지정할 수 있습니다.

```typescript
const str: string = "123";
const num: number = parseInt(str, 10); // 10진수
console.log(num); // 123
```

### 3. `parseFloat()` 함수 사용
`parseFloat()` 함수는 문자열을 부동소수점 숫자로 변환합니다.

```typescript
const str: string = "123.45";
const num: number = parseFloat(str);
console.log(num); // 123.45
```

### 4. `+` 연산자 사용
문자열 앞에 `+`를 붙여서 숫자로 변환할 수 있습니다.

```typescript
const str: string = "123";
const num: number = +str;
console.log(num); // 123
```

### 5. `Math.floor()`, `Math.ceil()`, `Math.round()` 등을 사용
소수점 아래 숫자를 다루기 위해서는 수학 함수를 사용할 수 있습니다.

```typescript
const str: string = "123.45";
const num: number = Math.floor(Number(str)); // 소수점 아래 버림
console.log(num); // 123
```

각 방법은 상황에 맞게 사용할 수 있으며, 문자열이 숫자로 변환될 수 없는 경우에는 `NaN`이 반환될 수 있으므로 이를 처리하는 로직이 필요할 수도 있습니다.



# DayJs

`dayjs`를 사용하여 현재 날짜에서 7일을 빼려면 `subtract` 메소드를 사용할 수 있습니다. 아래는 그 방법입니다:

```javascript
import dayjs from 'dayjs';

const date = dayjs().subtract(7, 'day'); // 현재 날짜에서 7일을 뺍니다.
console.log(date.format('YYYY-MM-DD'));  // 원하는 형식으로 출력
```

### 설명:
- `dayjs().subtract(7, 'day')`: 현재 날짜에서 7일을 뺀 날짜를 계산합니다.
- `.format('YYYY-MM-DD')`: 날짜를 원하는 형식으로 출력할 수 있습니다. (`YYYY-MM-DD` 형식은 예시입니다.)

이렇게 하면 현재 날짜에서 7일 전의 날짜를 얻을 수 있습니다.



# Table

### Table에서 라운드 적용 방법

현재 테이블에 `rounded-xl` 클래스를 적용하셨지만, `border-collapse` 속성이 적용된 상태에서는 라운딩이 제대로 동작하지 않을 수 있습니다. `border-collapse`는 테두리를 셀과 셀 사이에서 합치는 역할을 하므로, 라운드 모서리가 각 셀에 영향을 주지 않게 됩니다.

### 해결 방법:
1. **`border-collapse`를 제거**하거나,
2. **`overflow-hidden`과 함께 감싸는 컨테이너를 사용**하여 모서리 라운드를 적용하는 방법입니다.

#### 1. `border-collapse` 제거 후 적용
```html
<table class="mb-10 table-auto border-separate border rounded-xl w-[97%] mx-auto">
  <thead>
    <tr>
      <th class="px-4 py-2 bg-purple-200">Header 1</th>
      <th class="px-4 py-2 bg-purple-200">Header 2</th>
      <th class="px-4 py-2 bg-purple-200">Header 3</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td class="border px-4 py-2">Data 1</td>
      <td class="border px-4 py-2">Data 2</td>
      <td class="border px-4 py-2">Data 3</td>
    </tr>
    <tr>
      <td class="border px-4 py-2">Data 4</td>
      <td class="border px-4 py-2">Data 5</td>
      <td class="border px-4 py-2">Data 6</td>
    </tr>
  </tbody>
</table>
```
- `border-collapse` 대신 `border-separate`를 사용하면 라운딩이 적용될 수 있습니다.
  
#### 2. `overflow-hidden`으로 감싸기
```html
<div class="overflow-hidden rounded-xl">
  <table class="mb-10 table-auto border-collapse border w-[97%] mx-auto">
    <thead>
      <tr>
        <th class="px-4 py-2 bg-purple-200">Header 1</th>
        <th class="px-4 py-2 bg-purple-200">Header 2</th>
        <th class="px-4 py-2 bg-purple-200">Header 3</th>
      </tr>
    </thead>
    <tbody>
      <tr>
        <td class="border px-4 py-2">Data 1</td>
        <td class="border px-4 py-2">Data 2</td>
        <td class="border px-4 py-2">Data 3</td>
      </tr>
      <tr>
        <td class="border px-4 py-2">Data 4</td>
        <td class="border px-4 py-2">Data 5</td>
        <td class="border px-4 py-2">Data 6</td>
      </tr>
    </tbody>
  </table>
</div>
```
- 테이블을 감싸는 `<div>`에 `rounded-xl`과 `overflow-hidden`을 적용해서 테이블의 라운드 모서리를 시각적으로 나타냅니다.

이 방법을 시도하면 라운드 처리가 잘 적용될 것입니다.

### Table 너비 지정

Tailwind CSS를 사용하여 테이블에서 컬럼의 비율을 조정하려면, `table-layout` 속성을 활용할 수 있습니다. 특히 `table-layout: fixed`를 적용하면, 각 셀의 너비를 비율에 맞춰 고정할 수 있습니다. 그런 다음 각 컬럼에 Tailwind의 `w-` 클래스를 사용해 너비 비율을 설정할 수 있습니다.

### 예시: 컬럼 비율 설정
다음 예시는 3개의 컬럼에 각각 다른 비율로 너비를 설정한 것입니다.

```html
<table class="table-auto border-collapse w-[97%] mx-auto">
  <thead>
    <tr>
      <th class="w-1/4 px-4 py-2 bg-purple-200">Header 1 (25%)</th>
      <th class="w-1/2 px-4 py-2 bg-purple-200">Header 2 (50%)</th>
      <th class="w-1/4 px-4 py-2 bg-purple-200">Header 3 (25%)</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td class="border px-4 py-2">Data 1</td>
      <td class="border px-4 py-2">Data 2</td>
      <td class="border px-4 py-2">Data 3</td>
    </tr>
    <tr>
      <td class="border px-4 py-2">Data 4</td>
      <td class="border px-4 py-2">Data 5</td>
      <td class="border px-4 py-2">Data 6</td>
    </tr>
  </tbody>
</table>
```

### 주요 속성:
1. **`table-auto`**: 테이블이 자동으로 너비를 결정하는 기본 레이아웃입니다. 컬럼의 내용에 따라 크기가 결정됩니다.
2. **`w-1/4`, `w-1/2`**: Tailwind의 너비 클래스를 사용하여 컬럼의 비율을 지정합니다. 여기서는 각각 25%, 50%, 25%로 설정한 것입니다.
3. **`border-collapse`**: 셀 간의 테두리를 합쳐서 깔끔하게 보이도록 설정.

### 추가적으로 `table-layout: fixed`를 활용하는 방법:
만약 더 고정된 비율을 원하면, `table-layout: fixed`를 적용한 후 각 컬럼에 정확한 비율을 설정할 수 있습니다.

```html
<table class="table-fixed border-collapse w-[97%] mx-auto">
  <thead>
    <tr>
      <th class="w-1/4 px-4 py-2 bg-purple-200">Header 1</th>
      <th class="w-1/2 px-4 py-2 bg-purple-200">Header 2</th>
      <th class="w-1/4 px-4 py-2 bg-purple-200">Header 3</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td class="border px-4 py-2">Data 1</td>
      <td class="border px-4 py-2">Data 2</td>
      <td class="border px-4 py-2">Data 3</td>
    </tr>
    <tr>
      <td class="border px-4 py-2">Data 4</td>
      <td class="border px-4 py-2">Data
```



### Thead 없이 너비 지정

`thead` 없이 테이블을 구성하고, 각 `tr`에 `th` 대신 `td`로 스타일링하려면, 헤더와 바디 구분 없이 `tr`과 `td`만 사용해서 동일하게 컬럼 너비를 설정할 수 있습니다.

아래는 `thead` 없이 `td`를 사용한 예시입니다:

```html
<table class="table-fixed border-collapse w-[97%] mx-auto">
  <tr>
    <td class="w-1/4 px-4 py-2 bg-purple-200">Header 1</td>
    <td class="w-1/2 px-4 py-2 bg-purple-200">Header 2</td>
    <td class="w-1/4 px-4 py-2 bg-purple-200">Header 3</td>
  </tr>
  <tr>
    <td class="border px-4 py-2">Data 1</td>
    <td class="border px-4 py-2">Data 2</td>
    <td class="border px-4 py-2">Data 3</td>
  </tr>
  <tr>
    <td class="border px-4 py-2">Data 4</td>
    <td class="border px-4 py-2">Data 5</td>
    <td class="border px-4 py-2">Data 6</td>
  </tr>
</table>
```

### 설명:
1. **`table-fixed`**: 테이블을 고정 레이아웃으로 설정하여 각 컬럼의 너비를 미리 정의된 비율에 맞추어 적용합니다.
2. **`w-1/4`, `w-1/2`**: Tailwind의 너비 클래스를 사용하여 각 컬럼에 원하는 비율을 지정합니다.
3. **`bg-purple-200`**: 첫 번째 행을 헤더처럼 보이게 하기 위해 배경색을 지정합니다.

이 방식으로 `thead`를 사용하지 않고도 컬럼의 비율을 조정할 수 있습니다.

# Json 데이터 일괄 처리

아래와 같은 방식으로 타입스크립트에서 `JSON`을 처리할 수 있습니다. 각 탭을 클릭할 때 해당 키를 받아서 해당 값만 `true`로 설정하고 나머지는 `false`로 변경하도록 구현합니다.

```ts
type TestFlags = {
  [key: string]: boolean;
};

let testFlags: TestFlags = {
  Test1: true,
  Test2: false,
  Test3: false,
};

// 탭 클릭 시 호출되는 함수
function updateTestFlags(selectedKey: string) {
  Object.keys(testFlags).forEach((key) => {
    testFlags[key] = key === selectedKey;
  });
}

// 예시로 Test2 탭을 클릭했을 때
updateTestFlags('Test2');

console.log(testFlags); // 결과: { Test1: false, Test2: true, Test3: false }
```

### 설명:
1. `testFlags`라는 객체에 각 테스트의 상태(`true`/`false`)를 저장합니다.
2. `updateTestFlags` 함수는 클릭한 탭의 `key` 값을 받아, 해당 키는 `true`, 나머지는 `false`로 설정합니다.
3. `Object.keys(testFlags)`로 `testFlags`의 모든 키를 가져와서 각각 확인하며, 클릭한 탭의 키와 일치하는 경우만 `true`로 설정합니다.



# Vite와 Vue CLI의 차이

`Vite`와 `Vue CLI`(기본 Vue 프로젝트)를 사용하여 Vue 애플리케이션을 실행하는 방법에는 몇 가지 차이점이 있습니다. 주로 빌드 도구와 개발 서버의 동작 방식에 따른 차이가 발생하며, 각각의 방식은 프로젝트 초기 설정과 실행 방법에서 조금씩 다르게 동작합니다.

### 1. Vite를 사용한 Vue 실행 방법

`Vite`는 빠른 개발 서버와 빌드 프로세스를 제공하는 최신 도구로, 특히 모듈이 필요한 순간에만 빌드하는 방식(HMR, Hot Module Replacement)을 사용하여 빠른 개발 경험을 제공합니다.

#### Vite로 Vue 프로젝트 실행하는 방법
1. **Vite 프로젝트 초기화**:
   - Vite로 프로젝트를 생성할 때, `create-vite` 명령어를 사용하여 Vue 프로젝트를 생성합니다.

   ```bash
   npm create vite@latest my-vue-app --template vue
   ```

2. **패키지 설치**:
   - 프로젝트 디렉토리로 이동한 후, 필요한 패키지를 설치합니다.

   ```bash
   cd my-vue-app
   npm install
   ```

3. **개발 서버 실행**:
   - Vite의 개발 서버를 실행합니다. 이 서버는 빠르고, 코드 변경 시 빠르게 반영됩니다.

   ```bash
   npm run dev
   ```

   - 기본적으로 `localhost:5173`에서 프로젝트가 실행됩니다.

4. **프로덕션 빌드**:
   - 애플리케이션을 프로덕션용으로 빌드할 때는 다음 명령어를 사용합니다.

   ```bash
   npm run build
   ```

   - 빌드된 결과물은 `dist/` 폴더에 저장됩니다.

5. **빌드된 파일 미리보기**:
   - 빌드된 결과물을 미리보기 위해 Vite의 미리보기 서버를 실행할 수 있습니다.

   ```bash
   npm run preview
   ```

---

### 2. Vue CLI를 사용한 기본 Vue 실행 방법

Vue CLI는 전통적인 Vue 프로젝트를 위한 설정 도구로, 프로젝트 초기 설정에서 웹팩(Webpack) 기반으로 빌드 프로세스를 설정합니다. Vue CLI는 기존 프로젝트에서의 확장성과 다양한 설정 옵션을 제공합니다.

#### Vue CLI로 Vue 프로젝트 실행하는 방법
1. **Vue CLI 설치**:
   - 먼저, Vue CLI가 글로벌로 설치되어 있지 않다면 다음 명령어로 설치합니다.

   ```bash
   npm install -g @vue/cli
   ```

2. **Vue 프로젝트 생성**:
   - Vue CLI로 새로운 Vue 프로젝트를 생성합니다.

   ```bash
   vue create my-vue-app
   ```

3. **패키지 설치 및 프로젝트 시작**:
   - 프로젝트 디렉토리로 이동한 후, 다음 명령어로 개발 서버를 실행합니다.

   ```bash
   cd my-vue-app
   npm run serve
   ```

   - 기본적으로 `localhost:8080`에서 프로젝트가 실행됩니다.

4. **프로덕션 빌드**:
   - 프로덕션 환경을 위한 빌드는 다음 명령어를 사용합니다.

   ```bash
   npm run build
   ```

   - 빌드된 결과물은 `dist/` 폴더에 저장됩니다.

---

### 주요 차이점

| 항목                | Vite                               | Vue CLI                                       |
| ------------------- | ---------------------------------- | --------------------------------------------- |
| **빌드 도구**       | ESBuild, Rollup                    | Webpack                                       |
| **개발 서버**       | 매우 빠른 개발 서버                | 상대적으로 느린 웹팩 개발 서버                |
| **HMR**             | 즉시 반영 (더 빠름)                | HMR 지원, 하지만 속도는 느림                  |
| **프로젝트 초기화** | `npm create vite@latest`           | `vue create`                                  |
| **프로덕션 빌드**   | 더 작은 번들 크기와 빠른 빌드 속도 | 빌드 속도가 비교적 느림                       |
| **설정 가능성**     | 더 간단한 설정 (`vite.config.js`)  | 고도로 구성 가능 (`vue.config.js`, 웹팩 설정) |

Vite는 최신 도구로 속도와 간편함을 추구하고, Vue CLI는 다양한 설정과 확장성을 중점으로 두고 있습니다.
